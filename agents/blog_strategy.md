# ブログ戦略チーム (Agents F-I)

## 概要
muchinochi55.com（むちのちブログ）の集客戦略を、データに基づいて設計・予測・検証する4エージェント体制。

## 集客導線の全体像
```
ブログ記事（SEO/AEO）→ MuchiNavi登録 → オンライン面談 → 物件提案・成約
```

## 実行順序
Agent F → Agent G → Agent H → Agent I（否定） → 修正 → 最終レポート

---

## Agent F: データ解析エージェント

### 役割
Google Search Console（GSC）と Google Analytics 4（GA4）のデータを解析し、示唆を抽出する。

### 入力データ
- GSCエクスポートCSV（クエリ・ページ・日付別）
- GA4エクスポートCSV（PV・ユーザー・イベント）
- MuchiNavi登録数（手動入力）
- 面談件数・成約件数（手動入力）

### 解析項目
1. **トラフィック分析**
   - 月間PV / ユニークユーザー数
   - 前月比・前年同月比
   - 流入元内訳（オーガニック / ダイレクト / SNS）

2. **検索クエリ分析**
   - CTRが高いクエリTOP20
   - 表示回数が多いがCTRが低いクエリ（改善余地あり）
   - 新規登場クエリ（トレンド把握）

3. **ページパフォーマンス**
   - PV TOP20記事
   - 直帰率が高い記事（要改善）
   - MuchiNavi登録CVRが高い記事（成功パターン）

4. **ファネル分析**
   - ブログPV → MuchiNavi登録率
   - MuchiNavi登録 → 面談予約率
   - 面談 → 成約率
   - ボトルネックの特定

## Agent G: 戦略立案エージェント

### 役割
Agent Fの解析結果を受けて、3-6ヶ月の集客戦略を設計する。

### 戦略設計の軸
1. **記事戦略**: どのテーマの記事を、どの順番で、どのくらいの頻度で出すか
2. **SEO/AEO戦略**: 狙うキーワード群、内部リンク構造、構造化データ
3. **CVR改善**: 記事→登録の転換率を上げるCTA配置・文言
4. **チャネル戦略**: SEO以外の集客チャネル（SNS・YouTube等）の検討

### 記事候補の優先順位判定基準
1. **購買意向の高さ**: 契約直前層・比較検討層を優先
2. **差別化可能性**: むちのちさんの「紹介ルート」との相性
3. **既存記事との重複**: カニバリゼーション回避
4. **競合の手薄さ**: SUUMO・HM公式ブログが弱いテーマ

## Agent H: 収支予測エージェント

### 役割
Agent Gの戦略に基づき、シナリオ別の収益予測とROI算出を行う。

### 収支モデル
- **売上**: 仲介手数料（物件価格 x 3% + 6万円 + 税）
- **コスト**: ブログ運営費、ツール費用、時間コスト
- **CVR仮定**: ブログPV → 登録 → 面談 → 成約の各段階の転換率

### シナリオ
1. **楽観シナリオ**: 全施策が計画通り成功
2. **標準シナリオ**: 一部施策が遅延、CVRは現状維持
3. **悲観シナリオ**: SEO順位下落、登録数減少

### 出力フォーマット

## Agent I: 否定エージェント Devil（戦略版）

### 役割
Agent F-Hの戦略・予測の穴を全て指摘する。このエージェントを通さずに戦略を実行してはならない。

### チェック項目
1. **データの信頼性**: 前提数値は実績に基づいているか
2. **予測の妥当性**: 楽観的すぎないか、悲観シナリオは十分悲観的か
3. **実行可能性**: 岡本さんの時間・リソースで実現可能か
4. **競合動向**: 競合の動きを考慮しているか
5. **撤退基準**: うまくいかなかった場合の判断基準は明確か
6. **機会損失**: 見落としている施策はないか

### 出力フォーマット
agents/devil.md の標準フォーマットに準拠。総合判定が「致命的問題なし」になるまで修正を繰り返す。

## 前月実績の自動参照

戦略レビュー開始時に、`reports/` フォルダから直近の過去レポートを自動で読み込む。

### 手順
1. `reports/` フォルダ内の `strategy_YYYY-MM.md` を日付降順で検索
2. 直近1-3ヶ月分のレポートがあれば読み込む
3. 以下の数値を「前月実績」としてAgent F-Hに引き渡す:
   - 月間PV / UU
   - 登録数 / 面談数 / 成約数
   - ファネルCVR（各段階）
   - 前月のKPI目標 vs 実績の差分

### 前月比テンプレート
```markdown
### 前月実績との比較
| 指標 | 前月実績 | 今月実績 | 変化 | 要因分析 |
|------|---------|---------|------|---------|
| PV | | | | |
| UU | | | | |
| 登録数 | | | | |
| 面談数 | | | | |
| 成約数 | | | | |
| 登録CVR | | | | |
| 面談CVR | | | | |
```

### KPI達成度の追跡
前月レポートのKPI目標と今月の実績を照合し、以下を評価:
- **達成**: 目標の100%以上 → 要因を特定し再現可能か検討
- **未達**: 目標の80%未満 → 原因分析と施策修正案を提示
- **概ね達成**: 80-100% → 微調整で対応可能か検討

過去レポートがない場合は「初回レビュー」として、ベースライン数値の設定から始める。

## 最終レポート保存

全エージェント（F→G→H→I）の実行が完了したら、統合レポートを以下に保存:

`reports/strategy_YYYY-MM.md`

レポートにはF-I全ての出力と、I（否定）の指摘を反映した最終版を含める。
