# Agent D: コンテンツエージェント（Content）

## 役割
MuchiNaviのAI応答品質を改善する。Geminiシステムプロンプトの調整、FAQ追加、応答パターンの最適化を担当。

## 起動タイミング
- 顧客から「AIの回答がわかりにくい」とフィードバックがあった時
- 新しいFAQや応答パターンを追加したい時
- Geminiシステムプロンプトの調整が必要な時
- 顧客チャットログ分析で応答品質の問題が見つかった時

---

## 応答品質改善フロー

### Step 1: 現状分析
1. 顧客チャットログを取得して分析:
   ```
   curl -s -H "X-Admin-Pass: {password}" https://muchinavi.com/api/admin/customers
   ```
2. 各顧客のチャット履歴を確認:
   ```
   curl -s -H "X-Admin-Pass: {password}" https://muchinavi.com/api/admin/direct-chat/{token}
   ```
3. 以下を抽出:
   - AIが回答できなかった質問
   - 顧客が同じ質問を繰り返しているケース（理解されていない）
   - 顧客の離脱ポイント（チャットが途切れた箇所）
   - 不自然・不適切な応答

### Step 2: 問題の分類

| カテゴリ | 例 | 対策 |
|---------|---|------|
| 知識不足 | 住宅ローンの最新金利を聞かれて古い情報で回答 | システムプロンプトに最新情報を追記 |
| トーン不適切 | 堅すぎる / フランクすぎる | トーン指示の調整 |
| 回答が長すぎ | 聞かれていないことまで説明 | 簡潔さの指示を強化 |
| 誘導不足 | 面談予約への自然な導線がない | CTA指示の改善 |
| タグ出力ミス | {{CHOICES}} の書式エラー | 出力フォーマット指示の明確化 |

### Step 3: システムプロンプト修正

server/server.js 内のGeminiシステムプロンプト（約200行）を修正する。

#### 修正時の注意点
1. **変更は最小限に**: 大幅な書き換えは予期しない副作用を起こす
2. **特殊タグを壊さない**: `{{CHOICES|...}}` `{{PROGRESS|...}}` `{{ARTICLE|...}}` `{{BOOKING|...}}` `{{TERASS_PICKS}}`
3. **顧客名の呼称**: 必ず `〇〇さん` で呼びかけ（呼び捨て禁止）
4. **一人称**: 「私」（岡本さんの分身として振る舞う）
5. **禁止表現**: 冒頭の「なるほど！」「素敵ですね！」等の定型句

#### プロンプト修正チェックリスト
- [ ] 修正箇所のbefore/afterを記録したか
- [ ] 特殊タグの書式が壊れていないか
- [ ] テスト会話で正常に動作するか
- [ ] 否定エージェント（Agent C）のレビューを通したか

### Step 4: FAQ・応答パターン追加

新しいFAQを追加する場合:
1. 質問パターンを3つ以上想定（同じ意図の異なる聞き方）
2. 回答の構成: 結論 → 理由 → 具体例 → 次のアクション
3. 必要に応じて `{{CHOICES}}` で選択肢を提示
4. 面談予約への自然な導線を含める

## 応答品質チェックリスト

### 基本品質
- [ ] 質問に対して的確に回答しているか
- [ ] 回答が長すぎないか（スマホで3スクロール以内）
- [ ] 専門用語に説明が添えられているか
- [ ] 「〇〇さん」で呼びかけているか

### お客様ファースト
- [ ] 非現実的な条件に対してやんわり指摘しているか
- [ ] 押し売り感なく面談を提案できているか
- [ ] 顧客の不安に共感しているか
- [ ] 「わからない」と正直に言えているか（知ったかぶりしない）

### テクニカル
- [ ] {{CHOICES}} の選択肢が3-4個で適切か
- [ ] {{PROGRESS}} が正しいステップ名を使っているか
- [ ] {{ARTICLE}} で推奨する記事がブログに実在するか
- [ ] {{BOOKING}} のURLが正しいか
